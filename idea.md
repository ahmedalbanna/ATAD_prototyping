## وثيقة احتياجات تطبيق عتاد ATAD (MVP)
![[F:\MVP\ATAD\logo.png]]

### 1. وصف النسخة الأولى (MVP)

تطبيق "عتاد" في نسخته الأولى هو منصة رقمية بسيطة تتيح للمستخدمين الأساسيين:
- عرض الأصول المتاحة للتأجير
- تنفيذ عملية التأجير بشكل أساسي
- تقديم طلب تأجير، متابعة الطلب، قبوله أو رفضه

الهدف: اختبار السوق والتحقق من قابلية الاستخدام قبل تطوير الميزات المتقدمة.

---

### 2. أنواع المستخدمين

**أ) المستأجر:**
- إنشاء حساب وتسجيل دخول
- تصفح والبحث عن الأصول
- عرض تفاصيل الأصل
- تقديم طلب تأجير
- متابعة حالة الطلب
- إتمام الدفع

**ب) المؤجر:**
- إنشاء حساب
- إضافة أصل للتأجير
- إدارة الأصول
- استلام الطلبات
- قبول أو رفض الطلبات
- متابعة الطلبات

**ج) الإدارة (Admin - نسخة بسيطة):**
- عرض المستخدمين
- إدارة الأصول
- متابعة الطلبات

---

### 3. الميزات الأساسية في MVP

- تسجيل الدخول باستخدام رقم الجوال (OTP)
- إنشاء حساب (مؤجر / مستأجر)
- إضافة أصول للتأجير
- عرض الأصول
- البحث والتصفية الأساسية
- عرض تفاصيل الأصل
- تقديم طلب تأجير
- قبول أو رفض الطلب من قبل المؤجر

---

### 4. الشاشات الأساسية

- شاشة البداية (Splash)
- تسجيل الدخول / إنشاء حساب
- الصفحة الرئيسية
- قائمة الأصول
- صفحة تفاصيل الأصل
- شاشة إنشاء طلب تأجير
- شاشة الطلبات
- شاشة إضافة أصل (للمؤجر)
- لوحة المؤجر
- لوحة تحكم الأدمن (ويب)

---

### 5. رحلة الاستخدام الأساسية

**رحلة المستأجر:**
تسجيل الدخول → تصفح الأصول → اختيار أصل → تحديد مدة التأجير → إرسال الطلب → انتظار الموافقة → إتمام الدفع → متابعة حالة الطلب

**رحلة المؤجر:**
تسجيل الدخول → إضافة أصل → استلام الطلبات → قبول أو رفض الطلب → متابعة الطلبات

---

### 6. نظام الطلبات

كل طلب يمر بالحالات التالية:
- قيد الانتظار
- تمت الموافقة
- مرفوض
- نشط (بعد الدفع)
- منتهي
- مكتمل (Completed)

---

### 7. العقد في النسخة الأولى

- عرض نص "شروط ثابت"
- موافقة المستخدم عبر Checkbox
**لا يشمل:** توقيع إلكتروني، إنشاء عقود ديناميكية

---

### 8. نظام الدفع في MVP

يمكن استخدام:
- دفع تجريبي (Mock Payment)
- أو تحويل بنكي مع رفع إيصال

---

### 9. قاعدة البيانات (مبسطة)

**Users:**
id, name, phone, role

**Assets:**
id, owner_id, title, description, price_per_day, city

**Bookings:**
id, asset_id, tenant_id, start_date, end_date, total_price, status

**Payments:**
id, booking_id, amount, status

---

### 10. واجهات البرمجة (API)

- Auth API
- Assets API
- Bookings API
- Payments API

---

### 11. المتطلبات التقنية

- تطبيق موبايل (Android و iOS)
- لوحة تحكم ويب بسيطة
- دعم اللغة العربية (RTL)
- نظام صلاحيات حسب نوع المستخدم
- تخزين سحابي للصور
- نظام بسيط لإدارة الحالات (State Management)

---

### 12. متطلبات التصميم (UI/UX)

- واجهة بسيطة وسهلة الاستخدام
- تنظيم واضح لعرض الأصول
- تصميم مشابه لتطبيقات التجارة مثل تطبيق جرير من حيث ترتيب المحتوى وسهولة التصفح
- Mobile First تصميم
- استخدام ألوان وخطوط مريحة

---

### 13. ما لا يشمل MVP (يتم تأجيله)

- التحقق من الهوية
- العقود الإلكترونية المتقدمة
- نظام التقييمات
- المحادثة داخل التطبيق
- إدارة النزاعات
- الإشعارات المتقدمة
- الدفع الإلكتروني المتكامل
- نظام الودائع والتأمين
- التقارير والتحليلات المتقدمة

---

### 14. الهدف من النسخة الأولى

إطلاق نسخة تشغيلية سريعة تمكن من:
- تسجيل المستخدمين
- إضافة الأصول
- تنفيذ عملية تأجير كاملة بشكل مبسط
- اختبار سلوك المستخدمين
- التحقق من جدوى المشروع
